tojiru

Bind any document into a quiet, static web reader.

PDF · CBZ · CB7 · CBR · DjVu — one folder you can host anywhere.

read one now ↓

Point tojiru at a document and it pre-renders every page, writes a tiny manifest, and ships a self-contained folder. No server. No client-side PDF engine. The browser does nothing but turn pages it already has.

For born-digital text PDFs the pages become crisp vector SVG — sharp at any zoom, and often smaller than the source file.

What it weighs

Output bundle size relative to the input. Lower is better. Measured on a real sample corpus.

×0.9
Vector text PDF
Crisp SVG, smaller than the source.
×1.1
Comics (CBZ/CB7/CBR)
Images copied as-is, plus thumbnails.
×6
DjVu
Rendered to lossless WebP. Scans inflate.
×10–34
Scanned / raster PDF
Page-images wrapped in SVG. The wrong fit.

tojiru is built for vector text. Feed it a born-digital PDF and the reader is light and razor-sharp. Feed it a scan and it bloats — for those, the format is the wrong tool, and it says so.

Formats

  • PDF one vector SVG per page poppler / mupdf
  • CBZ zip of images no extra dependency
  • CB7 7-Zip archive of images p7zip
  • CBR RAR archive of images no extra dependency
  • DjVu rendered page by page djvulibre

Comics work out of the box. PDF and DjVu need one system tool, detected at runtime — if it's missing, tojiru tells you which package to install.

Get it

git clone <repo> tojiru
cd tojiru && npm install && npm run build && npm link

tojiru book.pdf --out site/
# serve the folder, or open site/index.html
